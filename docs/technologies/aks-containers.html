AKS & Kubernetes Containers

Container orchestration, deployments, scaling, and observability on Azure Kubernetes Service

Dockerfile Basics

FROM mcr.microsoft.com/dotnet/aspnet:10-alpine
WORKDIR /app
COPY --from=builder /app/publish .
EXPOSE 80
ENTRYPOINT ["dotnet", "Api.dll"]

Kubernetes Deployment

apiVersion: apps/v1
kind: Deployment
metadata:
  name: refund-api
spec:
  replicas: 3
  selector:
    matchLabels:
      app: refund-api
  template:
    metadata:
      labels:
        app: refund-api
    spec:
      containers:
      - name: refund-api
        image: myregistry.azurecr.io/refund-api:latest
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"

Key Concepts

  • Pod: Smallest deployable unit (one or more containers)
  • Deployment: Declarative updates with rolling restarts
  • Service: Exposes pods as network endpoint
  • HPA: Horizontal Pod Autoscaler (scale by CPU/memory)
