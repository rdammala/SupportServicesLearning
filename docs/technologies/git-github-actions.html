Git & GitHub Actions

Version control workflows, branching strategies, and CI/CD automation

Git Workflow

# Create feature branch
git checkout -b feature/refund-approval

# Make commits
git add .
git commit -m "Add refund approval logic"

# Push to remote
git push origin feature/refund-approval

# Create PR, get review, merge
git checkout main
git pull origin main
git merge feature/refund-approval

Branching Strategy (Git Flow)

  • main: Production-ready, tagged releases
  • develop: Integration branch for features
  • feature/*: Individual feature work
  • hotfix/*: Emergency production fixes

GitHub Actions

Workflows triggered on events (push, PR, schedule)

name: CI/CD
on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Build
        run: dotnet build
      - name: Test
        run: dotnet test
