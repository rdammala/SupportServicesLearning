YAML & Azure Pipelines

CI/CD configuration as code: build, test, deploy stages with gates and approvals

Pipeline Anatomy

trigger:
  - main

pool:
  vmImage: 'ubuntu-latest'

stages:
  - stage: Build
    jobs:
      - job: BuildJob
        steps:
          - script: dotnet build
          - script: dotnet test

  - stage: Deploy
    condition: succeeded()
    jobs:
      - deployment: DeployProd
        environment: 'production'
        strategy:
          runOnce:
            deploy:
              - script: ./deploy.sh

Key concepts

  • Trigger: What branches start the pipeline
  • Stage: Logical grouping (Build, Test, Deploy)
  • Condition: When to run (succeeded(), failed(), always())
  • Approval: Manual gate between stages

Best Practices

  • ✅ Use variables for secrets (reference Azure Key Vault)
  • ✅ Gate production deployments with approvals
  • ✅ Cache dependencies to speed builds
  • ✅ Parallelize independent jobs
  • ❌ Hardcode secrets in YAML
  • ❌ Deploy directly on commit (always test first)
